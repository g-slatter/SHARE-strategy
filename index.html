SHARE - platform strategy

SHARE is a CSS business first, built to compound. The affiliate-CSS operation is the company today: effectively all of current revenue, and the source of the proprietary asset (commerce + conversion data across many merchants and channels). The evolution is from channel to network: turn the data the channel produces into products that each make the others stronger (advertisers, data, and network effects all compounding). We build for today (CSS revenue) and build what compounds. Most of what follows is direction, not yet today's business: ShareView is rolling out to clients slowly, with one client engaging on the fixed-margin model.

The core - today's revenue (CSS)

CSS business

Core revenue

The revenue engine. A managed affiliate-CSS service across 100+ campaigns. Generates effectively all of today's income - and the proprietary data asset every other strand is built on.

Detail
What it is
SHARE runs Google Shopping (CSS) campaigns for advertisers at scale. This is the company today and the reason the rest of the strategy is fundable.
Why it leads
Every other strand either improves this business (ShareView, CSS Brain) or monetises the data it produces (the expansion). It is the substrate, not a line item.
What's next
Protect and scale it. Confirm revenue/margin figures to anchor the all-hands narrative.

ShareView

Live

The dashboard CSS is delivered through: the data, made transparent, that wins client budget. The endgame is a Google CSS partner combining self-serve advantages (transparency, control, PPC alignment), affiliate advantages (we invest upfront, paid on success), and a fixed-margin model.

Detail
What it is
Web platform surfacing engine state per campaign, recommendation inbox, structural change registration, cap schedule editor. Recently extended with agentic insights and a sidecar LLM.
Where it stands
Production with internal users. Agentic insights live but needs prompt engineering; sidecar LLM details being gathered.
What's next
Roll out the fixed-margin transparency dashboard to win budget share; prompt-tune the insights agent; integrate the CSS Brain decision logic once live.

CSS Brain

App build pending

The decision logic that runs CSS. It sits between Altitude's pipeline and ShareView, recommending campaign changes for a human to approve and push. The differentiating IP, business-owned (formerly "CSS Monitor"; renamed because it now decides, not just monitors). Workbook today (v0.49, 117 campaigns); porting to a Python service.

Detail
What it is
Pipeline (Altitude) feeds clean data; the business-owned decision logic decides what to do. Three pieces in the Brain app: per-domain engines (growth / efficiency / defensive agents), a cross-domain orchestrator, and an MCP layer for conversational ops via Cowork.
Where it stands
Workbook mature and stable; app playbook complete (Python + FastAPI + Fly.io + Postgres). Build trigger fires after 10 stable engine-days; ~2 of 10 (June 2026).
What's next
Pull the trigger; port the campaign engine to parity. The decision logic stays with the business; Altitude owns the pipeline.

The expansion - new revenue from the same data

ShareMedia

Live - Landsec

A trusted retail media network for premium physical retail. The performance data and category intelligence from CSS become the credibility behind a retail media offer. Landsec won: the first external income.

Detail
What it is
SHARE's commerce and category intelligence (what sells, where, and to whom) informs media across premium physical retail in ways traditional agencies, lacking cross-merchant data, cannot. This is what ShareView unlocks.
Where it stands
Landsec won. Engagement scope, deliverables, and timeline being captured.
What's next
Land Landsec well. Define a repeatable offer; decide consultancy vs productised delivery (or both).

ShareContent

In test

A new content product for publishers. We test content featuring products from the brand's own range that match products we've seen sell across the wider platform - using our data to lift revenue per article.

Detail
What it is
We use SHARE's cross-merchant conversion data to choose which of a brand's products to feature in an article - currently in test to see if data-matching increases revenue per article.
Vs Linkby
Linkby sells clicks per article; we sell revenue per article. Publishers prefer outcome over input, and only SHARE has the conversion data to back the match.
What's next
Read the test results. Pick pilot publishers; decide commercial structure (rev-share, CPA, flat) and product form.

ShareAgent

Vision stage

SHARE's position in agent-led commerce: visibility, ownership and scale in a world where AI agents do the buying. A forward bet, earned by the chain rather than forced early.

Detail
What it is
As buying shifts to agents acting for shoppers and advertisers, SHARE wants to own a place in that flow rather than be disintermediated by it.
What it is not
Not an internal operator tool: the recommend / approve / push loop already lives in the CSS Brain decision logic. ShareAgent is the outward bet.
Why last
ShareView unlocks ShareMedia, which earns ShareAgent. The data and the retail network build the credibility that earns a defensible position.

The data flywheel

How the loop compounds. The core (the CSS business, run by the CSS Brain logic and sold through ShareView) generates today's revenue and, as a byproduct, proprietary conversion data across many merchants. The expansion is a chain, not a list: ShareView unlocks ShareMedia, which earns ShareAgent, with ShareContent shaped by what we know. Each product throws off more signal back into the asset. The three growth loops compound: every advertiser makes the next easier to win, every signal sharpens the data, every participant lifts the value of all the others. The loop only spins because the core funds and feeds it.
